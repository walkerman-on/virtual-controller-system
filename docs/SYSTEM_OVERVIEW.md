# Система виртуального контроллера - Обзор

## Описание системы

Система виртуального контроллера представляет собой комплексное решение для управления промышленным процессом с использованием PID-регуляторов, OPC UA сервера, базы данных PostgreSQL и Telegram бота для мониторинга.

## Архитектура системы

```
│   PostgreSQL    │    │   OPC UA Server │    │  Process Model  │
│   Database      │◄──►│   (4840)        │◄──►│   Simulation    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         ▲                       ▲                       ▲
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  Primary        │    │  Backup         │    │  Analytics      │
│  Controller     │◄──►│  Controller     │    │  Service (8080) │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         ▲                       ▲                       ▲
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐

└─────────────────┘    └─────────────────┘    └─────────────────┘
```

## Компоненты системы

### 1. База данных PostgreSQL
- **Порт**: 5432
- **Контейнер**: digital-twin-db
- **Назначение**: Хранение данных процесса, PID параметров, логов
- **Переменные**: DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD

### 2. OPC UA Сервер
- **Порт**: 4840
- **Контейнер**: opcua-server
- **Назначение**: Предоставление данных процесса через OPC UA протокол
- **Переменные**: OPCUA_SERVER_PORT, OPCUA_SERVER_ENDPOINT

### 3. Модель процесса
- **Контейнер**: process-model
- **Назначение**: Симуляция промышленного процесса (бак с жидкостью)
- **Переменные**: MODEL_UPDATE_INTERVAL

### 4. Основной контроллер
- **Контейнер**: pid-controller-primary
- **Назначение**: PID-регулирование процесса
- **Переменные**: CONTROLLER_MODE, OPCUA_SERVER_ENDPOINT

### 5. Резервный контроллер
- **Контейнер**: pid-controller-backup
- **Назначение**: Резервное PID-регулирование
- **Переменные**: CONTROLLER_BACKUP_MODE

### 6. Сервис аналитики
- **Порт**: 8080
- **Контейнер**: analytics-service
- **Назначение**: Анализ данных процесса
- **Переменные**: ANALYTICS_PORT, ANALYTICS_HOST

### 7. Telegram бот
- **Контейнер**: telegram-notification-bot
- **Назначение**: Уведомления и мониторинг через Telegram
- **Переменные**: TELEGRAM_BOT_TOKEN, ADMIN_CHAT_ID

### 8. Controller Watchdog
- **Контейнер**: controller-watchdog-service
- **Назначение**: Мониторинг и перезапуск контроллеров
- **Переменные**: WATCHDOG_INTERVAL, WATCHDOG_TIMEOUT

### 9. System Watchdog
- **Контейнер**: watchdog-system-service
- **Назначение**: Мониторинг всех контейнеров системы
- **Переменные**: TELEGRAM_BOT_TOKEN

## Конфигурация

### .env файл
Содержит все переменные окружения:
- Пароли и токены
- Порты и хосты
- Настройки подключения

### config.json файл
Содержит параметры системы:
- Параметры модели процесса
- PID коэффициенты
- OPC UA переменные

## Запуск системы

```bash
# 1. Настроить .env файл
cp env.example .env
# Отредактировать .env с вашими настройками

# 2. Запустить систему
docker-compose up -d

# 3. Проверить статус
docker-compose ps
```

## Мониторинг

- **Логи**: `docker-compose logs [service_name]`
- **Статус**: `docker-compose ps`
- **Telegram**: Отправьте `/start` боту для подписки на уведомления

## Порты

- **5432** - PostgreSQL
- **4840** - OPC UA Server
- **8080** - Analytics Service

## Переменные окружения

Все переменные настраиваются в `.env` файле. Основные:
- `TELEGRAM_BOT_TOKEN` - токен Telegram бота
- `DB_PASSWORD` - пароль базы данных
- `OPCUA_SERVER_PORT` - порт OPC UA сервера
- `ANALYTICS_PORT` - порт сервиса аналитики

# Расширенное логирование ошибок и критических ситуаций

## Обзор

В систему виртуального контроллера добавлено расширенное логирование для отслеживания ошибок и критических ситуаций в модели процесса и контроллере. Это позволяет операторам и инженерам своевременно выявлять проблемы и принимать меры по их устранению.

## Типы логируемых событий

### 🚨 Критические ошибки (CRITICAL)
- Переполнение или опустошение бака
- Некорректные значения уровня жидкости
- Критические ошибки в расчетах модели
- Неудачное восстановление состояния системы

### ❌ Ошибки (ERROR)
- Некорректные входные параметры
- Отрицательные значения потоков
- Некорректные значения клапана
- Ошибки подключения к OPC UA серверу
- Ошибки сохранения данных в БД

### ⚠️ Предупреждения (WARNING)
- Приближение к критическим значениям
- Аномальные значения параметров
- Большие ошибки управления
- Нестабильность процесса
- Критические значения открытия клапана

### 🔍 Отладочная информация (DEBUG)
- Детальные расчеты клапана
- Состояние давления и плотности
- Коэффициенты пропускной способности

## Логирование в модели процесса

### Бак (Tank)
- **Критические потоки**: Логирование при превышении разумных лимитов (>1000 м³/ч)
- **Переполнение/опустошение**: Критические сообщения при достижении границ
- **Предупреждения о заполнении**: При заполнении >90% или <10%
- **Высокое давление**: При давлении >50000 Па

### Клапан (Valve)
- **Некорректные значения**: Отрицательные или превышающие 100% значения открытия
- **Критические открытия**: <5% (застой) или >95% (перелив)
- **Значительные изменения**: При изменении открытия >10%
- **Аномальные параметры**: Высокое давление (>200000 Па) или плотность (>2000 кг/м³)

### Модель процесса (ProcessModel)
- **Некорректные входные данные**: None значения или значения вне диапазона
- **Критические состояния**: Отрицательные уровни или превышение высоты бака
- **Нестабильность**: Разница потоков >50 м³/ч
- **Критические уровни**: Заполнение <5% или >95%

## Логирование в контроллере

### PID-регулятор
- **Некорректные входные данные**: None значения или отрицательный временной шаг
- **Аномальные значения**: Уставка или значение процесса вне диапазона 0-10м
- **Большие ошибки**: Ошибка >2м (предупреждение), >1м (информация)
- **Переполнение интеграла**: При достижении лимита интегральной составляющей
- **Ограничение выхода**: При выходе за пределы 0-100%

### Универсальный контроллер
- **Проверка переменных**: Корректность значений SP, PV, OP
- **Критические ошибки управления**: Ошибка >1м
- **Критические выходы**: <5% или >95%
- **Значительные изменения**: Отклонение клапана >30% от среднего значения
- **Ошибки подключения**: Проблемы с OPC UA сервером или БД

## Мониторинг без управления
- **Критические ошибки**: Ошибка >1м при мониторинге
- **Критические клапаны**: <5% или >95% при мониторинге
- **Недоступные значения**: При получении None значений

## Эмодзи для быстрой идентификации

- 🚨 - Критические ошибки
- ❌ - Ошибки
- ⚠️ - Предупреждения
- 🔍 - Отладочная информация
- 🔧 - Изменения настроек
- 📊 - Данные процесса
- 💧 - Потоки жидкости
- 🎛️ - Управление

## Настройка уровней логирования

Для изменения детальности логирования можно настроить уровень в коде:

```python
logging.basicConfig(
    level=logging.DEBUG,  # DEBUG, INFO, WARNING, ERROR, CRITICAL
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

## Тестирование логирования

Создан тестовый скрипт `test_logging.py` для проверки работы всех типов логирования:

```bash
python3 test_logging.py
```

Тест проверяет:
- Критические ситуации в баке
- Некорректные значения клапана
- Ошибки в модели процесса
- Критические сценарии работы системы

## Рекомендации по использованию

1. **Мониторинг в реальном времени**: Настройте систему мониторинга для отслеживания критических сообщений
2. **Алерты**: Настройте уведомления при появлении критических ошибок
3. **Анализ трендов**: Используйте логи для анализа стабильности процесса
4. **Диагностика**: Логи помогают быстро выявлять причины проблем

## Интеграция с системами мониторинга

Логи можно интегрировать с системами мониторинга:
- **ELK Stack** (Elasticsearch, Logstash, Kibana)
- **Grafana** с Loki
- **Prometheus** с Alertmanager
- **Splunk**

## Заключение

Расширенное логирование значительно улучшает наблюдаемость системы и помогает операторам быстро реагировать на критические ситуации. Система теперь предоставляет детальную информацию о состоянии процесса и позволяет предотвращать аварийные ситуации.

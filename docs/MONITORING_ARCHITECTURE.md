# 🎛️ Новая архитектура мониторинга системы

## 📋 Что было реализовано:

### 1. **Controller Watchdog Service** 🎛️
- **Назначение**: Специализированный сервис для мониторинга контроллеров
- **Функции**:
  - Мониторинг heartbeat контроллеров через OPC UA
  - Автоматическая перезагрузка контроллеров при сбоях
  - Отправка уведомлений о критических ситуациях
  - Отслеживание количества перезагрузок
- **Файлы**: `controller-watchdog-service/controller_watchdog.py`

### 2. **System Watchdog Service** 🔍
- **Назначение**: Мониторинг всех контейнеров системы
- **Функции**:
  - Проверка статуса всех контейнеров Docker
  - Мониторинг health check'ов
  - Автоматическая перезагрузка проблемных контейнеров
  - Сводные уведомления о состоянии системы
- **Файлы**: `watchdog-system-service/system_watchdog.py`

### 3. **Уведомления в модели процесса** 🏭
- **Критические условия**:
  - 🚨 Переполнение резервуара (>2.8м)
  - 🚨 Опорожнение резервуара (<0.1м)
  - 🚨 Клапан застрял в критическом положении
- **Предупреждения**:
  - ⚠️ Высокий уровень (2.4-2.8м)
  - ⚠️ Низкий уровень (0.1-0.3м)
- **Файлы**: Обновлен `model/model_client.py`

### 4. **Обновленная конфигурация Docker** 🐳
- Заменен старый `watchdog` на два новых сервиса
- Добавлен доступ к Docker socket для перезагрузки контейнеров
- Добавлены Telegram зависимости в модель процесса
- Настроены правильные зависимости между сервисами

## 🔧 Архитектурные улучшения:

### **Разделение ответственности**:
- **Controller Watchdog**: Только контроллеры и их перезагрузка
- **System Watchdog**: Общий мониторинг контейнеров
- **Process Model**: Собственные уведомления о критических условиях

### **Автоматическое восстановление**:
- Перезагрузка контроллеров при потере heartbeat
- Перезагрузка контейнеров при сбоях
- Предотвращение спама уведомлений

### **Централизованные уведомления**:
- Все сервисы используют общий Telegram модуль
- Синхронизация подписчиков через общий volume
- Время по МСК для всех уведомлений

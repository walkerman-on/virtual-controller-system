# Архитектура системы цифрового двойника процесса

## Обзор системы

Система цифрового двойника представляет собой микросервисную архитектуру для моделирования и управления технологическим процессом уровня жидкости в резервуаре. Архитектура построена на основе Docker контейнеров и протокола OPC UA для обеспечения надежного межсервисного взаимодействия.

## Схема взаимодействия компонентов

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   OPC UA Server │    │  Process Model  │    │ PID Controller  │
│                 │    │                 │    │                 │
│  ┌─────────────┐│    │  ┌─────────────┐│    │  ┌─────────────┐│
│  │ PV_level    ││◄───┤  │ Tank        ││    │  │ PID Logic   ││
│  │ SP_level    ││    │  │ Valve       ││    │  │             ││
│  │ OP_valve    ││◄───┼──┤             ││◄───┼──┤             ││
│  │ outlet_flow ││    │  │             ││    │  │             ││
│  │ inlet_flow  ││    │  └─────────────┘│    │  └─────────────┘│
│  └─────────────┘│    └─────────────────┘    └─────────────────┘
└─────────────────┘
```

## Поток данных

1. **SP (Setpoint)** - уставка уровня жидкости (1.6 м)
2. **PID Controller** читает SP и PV, вычисляет OP по ГОСТ формулам
3. **OP (Output)** - управляющее воздействие на клапан (0-100%)
4. **Process Model** читает OP, вычисляет новый PV по физическим законам
5. **PV (Process Value)** - текущий уровень жидкости
6. **Цикл повторяется** каждые 0.1 секунды

## Параметры модели

### Бак
- **Высота:** 3.0 м
- **Диаметр:** 2.0 м
- **Плотность жидкости:** 1000 кг/м³
- **Начальный уровень:** 1.5 м
- **Площадь поперечного сечения:** 3.14 м²

### Клапан
- **Начальное открытие:** 50%
- **Диапазон:** 0-100%
- **Kv0 (минимальный):** 0.01
- **Kvs (максимальный):** 800
- **Расчет по ГОСТ:** Q = (Kv / (3.57 × 10⁴)) × √(Δp / ρ)

### Потоки
- **Входной расход:** 100 м³/ч (постоянный)
- **Выходной расход:** 160-200 м³/ч при Kv100
- **Гидростатическое давление:** ρ × g × h

## Параметры PID-регулятора

### Настройки по ГОСТ
- **Kp (пропорциональный):** 15.0
- **Ki (интегральный):** 0.1
- **Kd (дифференциальный):** 1.0
- **Уставка:** 1.6 м
- **Выходной диапазон:** 0-100%
- **Ограничение интеграла:** 5.0

### Формулы расчета
- **Ошибка:** e = action × (PV - SP), где action = 1 (прямое действие)
- **Пропорциональная:** P = Kc × e
- **Интегральная:** I = (1/Tи) × Ii, где Ii = Ii-1 + ei × Δt
- **Выходной сигнал:** OP = Kc×e + (1/Tи)×Ii + ОРнач

## Временные характеристики

- **Шаг симуляции модели:** 0.1 с
- **Интервал обновления контроллера:** 0.1 с
- **Частота обновления OPC UA:** 10 Гц
- **Время стабилизации:** ~100-200 секунд

## Модульная архитектура

### Базовые классы

1. **ProcessComponent** - базовый класс для всех компонентов модели
2. **Tank** - модель резервуара с жидкостью
3. **Valve** - модель клапана с расчетом по ГОСТ
4. **PIDController** - регулятор с ГОСТ формулами

### Расширяемость

Система спроектирована для легкого расширения:

1. **Новые компоненты модели** - добавление через наследование от ProcessComponent
2. **Новые переменные OPC UA** - через конфигурацию в config.json
3. **Дополнительные контроллеры** - независимые контейнеры
4. **Горизонтальное масштабирование** - множественные экземпляры компонентов

## Отказоустойчивость

- **Независимость компонентов** - модель продолжает работу при недоступности контроллера
- **Централизованное управление** - OPC UA сервер обеспечивает единую точку данных
- **Изоляция контейнеров** - каждый компонент работает независимо
- **Автоматический перезапуск** - контейнеры перезапускаются при сбоях
- **Graceful shutdown** - корректное завершение работы при остановке

## Производительность

- **Асинхронное взаимодействие** - неблокирующие операции OPC UA
